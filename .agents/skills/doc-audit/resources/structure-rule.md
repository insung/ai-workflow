# 룰 파일 구조 체크리스트 (Rule Structure Checklist)

대상 파일이 룰(`GEMINI.md`, `.rules/**/*.md`)인 경우, 아래 필수 섹션을 순번대로 점검하라.

## 필수 섹션

| # | 섹션 | 요구사항 | 필수 여부 |
|---|------|---------|----------|
| 1 | 제목 및 적용 범위 | 이 룰이 무엇에 적용되는지 1~2문장으로 명시되어 있는가? | 필수 |
| 2 | 행동 규칙 | 에이전트가 따라야 할 지시사항이 명령문("~하라") 형태로 존재하는가? | 필수 |
| 3 | 금지 조항 | 에이전트가 하지 말아야 할 행동이 별도 섹션으로 명시되어 있는가? | 필수 |
| 4 | Output 포맷 | 에이전트 출력 형태에 제약이 있으면 Skeleton이 제시되어 있는가? | 해당 시 필수 |

## 생성 시 Skeleton

룰 파일을 새로 생성할 때 아래 구조를 기본 뼈대로 사용하라:

```markdown
# {룰 제목}

{이 룰의 적용 범위를 1~2문장으로 명시}

## Rule 1: {규칙명}
- {명령문 형태의 지시사항}

## Rule N: {규칙명}
- {명령문 형태의 지시사항}

## 금지 조항 (Negative Constraints)
- [DANGER] {금지 행동}
```
