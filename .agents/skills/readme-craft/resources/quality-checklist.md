# README 품질 체크리스트 (Quality Checklist)

생성된 README.md에 대해 아래 10개 항목을 순번대로 점검하라. 건너뛰기를 금지한다.

## 채점 기준

| # | 기준 | 점검 방법 | 배점 |
|---|------|----------|------|
| 1 | 한 줄 설명 | Title 아래에 10단어 이내 설명이 존재하는가? | /1 |
| 2 | Quick Start | 5개 이하 복사-붙여넣기 가능한 명령어가 존재하는가? | /1 |
| 3 | 코드 예시 | 동작하는 코드 예시가 1개 이상 존재하는가? | /1 |
| 4 | 설치 옵션 | 2개 이상의 설치 방법이 제시되어 있는가? | /1 |
| 5 | 비주얼 | 스크린샷, GIF, 또는 아키텍처 다이어그램이 존재하는가? (TODO 플레이스홀더 허용) | /1 |
| 6 | 설정 테이블 | 설정 옵션이 테이블 형태로 정리되어 있는가? (해당 시) | /1 |
| 7 | 기여 가이드 | Contributing 섹션 또는 CONTRIBUTING.md 링크가 존재하는가? | /1 |
| 8 | 트러블슈팅 | 빈출 에러 + 해결법이 1개 이상 존재하는가? | /1 |
| 9 | 라이선스 | License 섹션이 존재하는가? | /1 |
| 10 | 플레이스홀더 검증 | 실제 API 키, 비밀번호, 내부 URL이 포함되어 있지 않은가? | /1 |

## 판정

| 점수 | 등급 | 조치 |
|------|------|------|
| 8-10 | PASS | 사용자에게 제시 |
| 6-7 | WARN | 미충족 항목을 명시하고 사용자에게 제시 |
| 0-5 | FAIL | 미충족 항목을 보완한 후 재검증 |
